interface RuntimePanelState {

## 17. Code panel spec

### 17.1 Responsibilities
- render file content read-only
- syntax highlight if possible
- scroll to line
- select single-line ranges
- highlight multi-line blocks

### 17.2 Data model
```ts
interface CodePanelState extends RuntimePanelState {
  type: "code";
  filePath?: string;
  language?: string;
  content?: string;
  currentLine?: number;
  selection?: {
    line: number;
    startCol: number;
    endCol: number;
  };
}
```

### 17.3 File loading
File reading occurs in Electron main process via IPC.

Required IPC method:
- `fs:readTextFile(path)`

Response:
```ts
interface ReadTextFileResult {
  path: string;
  content: string;
  exists: boolean;
}
```

### 17.4 Supported files
v1 assumes plain text files.

Binary or oversized files:
- reject with validation/runtime error

Suggested max file size v1:
- 2 MB soft limit
- warning above 500 KB

## 18. Browser panel spec

### 18.1 Responsibilities
- display webpage
- navigate to URL
- expose controlled automation helpers
- detect visible text
- click text-matched controls
- type into identified inputs
- return structured failure info

### 18.2 Data model
```ts
interface BrowserPanelState extends RuntimePanelState {
  type: "browser";
  currentUrl?: string;
  title?: string;
  loading: boolean;
}
```

### 18.3 Embedded browser
Use `<webview>`.

Required properties:
- preload script path
- sandboxed config where possible
- node integration disabled
- context isolation enabled

### 18.4 Preload API
Expose a narrow automation bridge from preload to guest page.

Example API surface:
```ts
window.__smh = {
  findClickableByText,
  clickByText,
  typeByLabelOrPlaceholder,
  hasVisibleText,
  highlightText,
  pressKey,
};
```

The renderer should not directly inject arbitrary JavaScript strings generated by AI.

### 18.5 DOM automation rules
For `click_text`:
1. gather clickable candidates
2. compute visible text
3. exact match first
4. case-insensitive match second
5. substring unique match third
6. if multiple equally good matches, throw ambiguity error

For `type_text`:
1. try `<label for>`
2. try enclosing `<label>`
3. try `aria-label`
4. try `placeholder`
5. try `name`
6. try nearby text heuristic
7. ambiguity -> error

### 18.6 Timeouts
Default browser command timeout:
- 5000 ms

Navigation timeout:
- 10000 ms

### 18.7 Settling
After click/type actions, add a small settle delay:
- 150 ms default

## 19. TTS subsystem

### 19.1 Engine
Use `window.speechSynthesis`.

### 19.2 Requirements
- queue one utterance at a time
- resolve on `onend`
- reject on `onerror`
- support voice selection if available
- support rate from metadata if available

### 19.3 Data model
```ts
interface TtsConfig {
  voice?: string;
  rate?: number;
  pitch?: number;
  volume?: number;
}
```

### 19.4 Fallback behavior
If TTS unavailable:
- show warning
- still allow playback
- resolve `tts()` after 300 ms with warning logged

## 20. Layout manager

### 20.1 v1 layouts
- `single`: one visible panel full width
- `two-column`: two visible columns, panels flow left to right
- `grid`: 2x2 style region for up to 4 visible panels

### 20.2 Rules
- more panels than visible slots should still exist but only active/focused panels are shown based on insertion order for v1, or all panels scroll in a workspace area

Recommendation:
For v1, simplest approach is a responsive workspace where all open panels render and CSS controls the layout.
